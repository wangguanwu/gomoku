# Gomoku 科幻炫酷效果优化方案

## 1. 项目现状

- **技术栈**: Vanilla JS + HTML5 Canvas (无框架、无构建工具)
- **渲染方式**: 每次状态变化重绘整个画布
- **视觉效果**: 仅基础阴影和高光，无动画

---

## 2. 视觉效果设计

### 2.1 配色方案 (Cyberpunk/Sci-Fi)
| 元素 | 颜色 | 说明 |
|------|------|------|
| 背景 | `#0a0a0f` | 深空黑 |
| 棋盘 | `#1a1a2e` | 深紫蓝 |
| 网格线 | `#00ffff` (cyan) | 霓虹青色，发光效果 |
| 黑棋 | `#ff00ff` (magenta) | 品红，带发光 |
| 白棋 | `#00ffff` (cyan) | 青色，带发光 |
| 强调色 | `#ffff00` (yellow) | 获胜提示 |
| 辅助色 | `#7b2cbf` (purple) | 次要元素 |

### 2.2 粒子系统
- **落子动画**: 粒子爆发效果，从落点向外扩散
- **获胜动画**: 五子连线高亮 + 粒子流动

### 2.3 动画效果
| 动画 | 描述 | 时长 |
|------|------|------|
| 落子缩放 | 棋子从0缩放到1，带弹性缓动 | 300ms |
| 落子光晕 | 棋子出现时带有脉冲光环 | 500ms |
| 网格呼吸 | 网格线微弱呼吸/脉动效果 | 循环 |
| 悬停高亮 | 鼠标悬停格子时边框发光 | 即时 |
| 获胜闪烁 | 获胜五子颜色交替闪烁 | 1000ms |

## 3. 实施方案

### 3.1 目录结构
```
gomoku/
├── index.html        # HTML结构 (保持不变)
├── style.css         # 样式 (重大更新)
├── script.js         # 游戏逻辑 (重大更新)
└── effects.js        # 新增: 粒子系统和特效
```

### 3.2 CSS 改动 (`style.css`)
- 深色主题背景
- 霓虹发光效果 (`box-shadow`, `text-shadow`)
- 按钮渐变 + hover 动画
- 棋盘容器发光边框

### 3.3 粒子系统 (`effects.js`)
- `Particle` 类: 管理单个粒子
- `ParticleSystem` 类: 管理粒子池
- `createDropParticles(x, y, color)`: 落子爆发
- `createWinParticles(line)`: 获胜粒子

### 3.4 Canvas 渲染改动 (`script.js`)
- 重构 `drawPiece()`: 支持动画和发光
- 新增 `drawGlow()`: 绘制发光效果
- 新增 `drawGridPulse()`: 网格呼吸效果
- 新增 `requestAnimationFrame` 循环: 驱动动画
- 新增 `drawHoverCell()`: 悬停高亮

### 3.5 性能考虑
- 使用 `requestAnimationFrame` 统一管理动画
- 粒子数量限制 (每帧最多 100 个)
- Canvas 状态保存/恢复优化
